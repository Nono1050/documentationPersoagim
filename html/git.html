Documentation rapide de GIT

N'hésitez pas a utiliser Ctrl-F pour aller chercher un text directement

Lors de projet commun

Etape 1 : Creer un fork du projet, sur le github du chef de projet. Cela va creer une copie sur votre github

Etape 2 : Ouvrez le terminal Bash a l'endroit ou vous voulez cloner votre projet et entrez la commande
git clone (clé SSH)

Etape 3 : Vous devez alors vérifié si vous etes connecter à l'upstream avec git remote -v

Cas n°1

dans le cas ou vous n'êtes pas connecter, vous devez entrée la commande git remote add upstream (clé ssh) pour lié l'upstream avec l'origin (votre arbre(origin) avec l'arbre du chef de projet(upstream))

Cas n°2

Vous êtes connecter à l'upstream ? Vous pouvez passer à l'étape suivante

Etape 4 : Un fois connecter a l'upstream, vous devez vérifié si votre main est a jour avec la main de l'upstream

Cas n°1

Si votre main n'est pas a jour avec l'uptstream main, entrée le code git pull upstream main pour récupéré les modification, suivi d'un git push origin main pour envoyer ce que vous avez récupéré sur votre main

Cas n°2

Si votre main est a jour, passer directement à l'étape suivante

Etape 5 : Une fois votre main mis a jour, entrée la commande git checkout -b nomDeVotreNouvelleBranche pour creer une nouvelle branche et vous placer dessu. A ce moment la, commencez vos travaux.
(chaque fois que vous voulez faire un nouveau travail, vous devez a chaque fois créer une nouvelle branche et ne pas utiliser les branches existante.)

Etape 6 : Une fois vos travaux fini, faite un git add . suivi d'un git commit -m "Message de votre commit" et git push origin nomDeVotreNouvelleBranche pour enregistrer votre travail sur votre nouvelle branche.

Etape 7 : Une fois que votre travail est push sur votre nouvelle Branche. Vous devez faire un pull request sur votre fork de github, cela fera automatiquement une demande au chef de projet sur son github

Etape 8 : Une fois que vous avez cliqué sur new pull request. Vous devez annoncer de ou vers ou vous voulez creer le pull request. Ici vous partez de votre nomDeVotreNouvelleBranche vers la branche main du projet original. Et vous envoyer le pull request
